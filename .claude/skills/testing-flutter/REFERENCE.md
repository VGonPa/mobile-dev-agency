# Testing Flutter — Implementation Reference

Full test templates for patterns described in [SKILL.md](SKILL.md). Every template here corresponds to a decision made in SKILL.md — don't copy-paste without reading the strategy context first.

## Service / Use Case Test Template

**When needed:** Testing business logic classes that depend on repositories.
**Mock boundary:** Repository level (see SKILL.md → Mock Boundary Strategy).

```dart
import 'package:flutter_test/flutter_test.dart';
import 'package:mocktail/mocktail.dart';

class MockUserRepository extends Mock implements UserRepository {}
class FakeUser extends Fake implements User {}

void main() {
  late UserService service;
  late MockUserRepository mockRepo;

  setUpAll(() {
    registerFallbackValue(FakeUser());
  });

  setUp(() {
    mockRepo = MockUserRepository();
    service = UserService(mockRepo);
  });

  group('UserService', () {
    group('getUser', () {
      test('should return user when found', () async {
        // Arrange
        when(() => mockRepo.getUser('123'))
            .thenAnswer((_) async => testUser);

        // Act
        final result = await service.getUser('123');

        // Assert
        expect(result, equals(testUser));
      });

      test('should throw when user not found', () async {
        when(() => mockRepo.getUser('999'))
            .thenThrow(UserNotFoundException());

        expect(
          () => service.getUser('999'),
          throwsA(isA<UserNotFoundException>()),
        );
      });
    });

    group('deleteUser', () {
      test('should call repository delete when user exists', () async {
        // When the SIDE EFFECT is the behavior, verify is appropriate
        when(() => mockRepo.deleteUser(any()))
            .thenAnswer((_) async {});

        await service.deleteUser('123');

        verify(() => mockRepo.deleteUser('123')).called(1);
      });
    });
  });
}
```

## Model Test Template

**When needed:** Models with custom logic (computed properties, validation, non-trivial `fromJson`).
**When NOT needed:** Models generated by `freezed` or `json_serializable` with no custom methods.

```dart
group('Product', () {
  test('fromJson creates Product correctly', () {
    final json = {'id': '1', 'name': 'Test', 'price': 9.99};
    final product = Product.fromJson(json);

    expect(product.id, '1');
    expect(product.name, 'Test');
    expect(product.price, 9.99);
  });

  test('copyWith creates new instance with updated values', () {
    final product = Product(id: '1', name: 'Test', price: 9.99);
    final updated = product.copyWith(price: 19.99);

    expect(updated.price, 19.99);
    expect(product.price, 9.99); // Original unchanged — immutability check
  });

  test('discountedPrice applies percentage correctly', () {
    // Only test custom logic methods — not generated code
    final product = Product(id: '1', name: 'Test', price: 100.0);
    expect(product.discountedPrice(0.2), 80.0);
  });
});
```

## Widget Test Templates

### Basic Widget Test

**When needed:** Testing that a widget renders correct data and responds to interactions.

```dart
testWidgets('ProductCard displays product info', (tester) async {
  final product = Product(id: '1', name: 'Widget', price: 9.99);

  await tester.pumpWidget(
    MaterialApp(
      home: Scaffold(body: ProductCard(product: product)),
    ),
  );

  expect(find.text('Widget'), findsOneWidget);
  expect(find.text('\$9.99'), findsOneWidget);
});

testWidgets('ProductCard tap triggers callback', (tester) async {
  bool tapped = false;

  await tester.pumpWidget(
    MaterialApp(
      home: Scaffold(
        body: ProductCard(
          product: testProduct,
          onTap: () => tapped = true,
        ),
      ),
    ),
  );

  await tester.tap(find.byType(ProductCard));
  await tester.pump(); // One frame to process the tap

  expect(tapped, true);
});
```

### Widget Test with Riverpod Overrides

**When needed:** Widgets that read from Riverpod providers.

```dart
testWidgets('UserPage shows loading then user name', (tester) async {
  final mockRepo = MockUserRepository();
  when(() => mockRepo.getCurrentUser())
      .thenAnswer((_) async => testUser);

  await tester.pumpWidget(
    ProviderScope(
      overrides: [
        userRepositoryProvider.overrideWithValue(mockRepo),
      ],
      child: const MaterialApp(home: UserPage()),
    ),
  );

  // Initial state: loading
  expect(find.byType(CircularProgressIndicator), findsOneWidget);

  // After async completes
  await tester.pump(); // Let the Future complete
  await tester.pump(); // Rebuild with new state

  expect(find.text(testUser.name), findsOneWidget);
});
```

### Test Helper: pumpApp Extension

**Why:** Eliminates `MaterialApp` boilerplate in every widget test.

```dart
// test/helpers/pump_app.dart
extension WidgetTesterX on WidgetTester {
  Future<void> pumpApp(Widget widget, {List<Override> overrides = const []}) async {
    await pumpWidget(
      ProviderScope(
        overrides: overrides,
        child: MaterialApp(home: widget),
      ),
    );
  }
}
```

## Riverpod Controller Test Template

**When needed:** Testing Notifier/AsyncNotifier state transitions.
**Mock boundary:** Service or repository level, depending on controller complexity.

```dart
void main() {
  late MockCartService mockService;

  setUp(() {
    mockService = MockCartService();
  });

  test('addItem updates cart state', () async {
    when(() => mockService.addToCart(any()))
        .thenAnswer((_) async => updatedCart);

    final container = ProviderContainer(
      overrides: [
        cartServiceProvider.overrideWithValue(mockService),
      ],
    );
    addTearDown(container.dispose);

    // Read initial state
    final notifier = container.read(cartControllerProvider.notifier);

    // Act
    await notifier.addItem(testItem);

    // Assert state transition
    final state = container.read(cartControllerProvider);
    expect(state.items, contains(testItem));
  });
}
```

## Integration Test Template

**When needed:** Critical user flows that span multiple screens (login, purchase, onboarding).
**Limit:** 3-5 integration tests max. They're slow and fragile.

```dart
// integration_test/app_test.dart
import 'package:integration_test/integration_test.dart';

void main() {
  IntegrationTestWidgetsFlutterBinding.ensureInitialized();

  testWidgets('login and browse flow', (tester) async {
    app.main();
    await tester.pumpAndSettle();

    // Login
    await tester.enterText(find.byKey(Key('email_field')), 'test@example.com');
    await tester.enterText(find.byKey(Key('password_field')), 'password123');
    await tester.tap(find.byKey(Key('login_button')));
    await tester.pumpAndSettle();

    // Verify navigation to home
    expect(find.text('Welcome'), findsOneWidget);
  });
}
```

## Golden Test Template

**When needed:** Design-system components where visual regression matters.
**When NOT needed:** Most app screens — too brittle, breaks on font rendering differences across OS versions.

```dart
testWidgets('ProductCard matches golden', (tester) async {
  await tester.pumpWidget(
    MaterialApp(
      home: Scaffold(body: ProductCard(product: testProduct)),
    ),
  );

  await expectLater(
    find.byType(ProductCard),
    matchesGoldenFile('goldens/product_card.png'),
  );
});
```

```bash
# Update golden files after intentional visual changes
flutter test --update-goldens
```

## Shared Mock Definitions

**Why centralize:** Avoids duplicate mock class definitions across test files. Define once, import everywhere.

```dart
// test/mocks/mocks.dart
import 'package:mocktail/mocktail.dart';

// Repository mocks
class MockUserRepository extends Mock implements UserRepository {}
class MockAuthRepository extends Mock implements AuthRepository {}
class MockCartRepository extends Mock implements CartRepository {}

// Service mocks (for controller tests only)
class MockUserService extends Mock implements UserService {}
class MockCartService extends Mock implements CartService {}

// Fakes for any() matchers
class FakeUser extends Fake implements User {}
class FakeProduct extends Fake implements Product {}
```

```dart
// test/mocks/mocks_setup.dart
void registerAllFallbackValues() {
  registerFallbackValue(FakeUser());
  registerFallbackValue(FakeProduct());
}

// Usage in test files:
// setUpAll(() => registerAllFallbackValues());
```

## Test Fixtures / Factories

**Why factories over inline data:** Test data defined once, consistent across all tests. Changes to model structure require updates in one place.

```dart
// test/fixtures/test_data.dart
final testUser = User(
  id: '123',
  name: 'Test User',
  email: 'test@example.com',
);

final testProduct = Product(
  id: '1',
  name: 'Test Product',
  price: 9.99,
);

// For tests that need variations:
User makeUser({String? id, String? name}) => User(
  id: id ?? '123',
  name: name ?? 'Test User',
  email: 'test@example.com',
);
```
